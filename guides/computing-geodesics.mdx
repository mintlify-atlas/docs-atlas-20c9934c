LIgeodesics provides two complementary approaches to computing geodesics:

- **Closed-form functions** — specialized implementations for SE(n) and SIM(n) that exploit the group structure to avoid numerical integration.
- **Generic `Exp` / `Log`** — a Taylor-series integrator that works with any matrix Lie group defined by a basis of its algebra.

## Closed-form geodesics

For the groups SE(n) (rigid motions) and SIM(n) (similarities) the toolbox ships optimized closed-form implementations.

### SE(n) — special Euclidean group

`Exp_SE` computes the geodesic starting from the identity under a left-invariant metric. The velocity matrix `U0` must be structured as a Lie-algebra element of SE(n): a skew-symmetric rotation block and a translation column in the last column, with a zero bottom row.

```matlab
% Compute the endpoint at t = 1 (default)
Q1 = Exp_SE( U0 );

% Evaluate at multiple times along the geodesic
times = linspace(0, 1, 100);
Qt   = Exp_SE( U0, times );   % Qt is (d+1) x (d+1) x 100
```

To also retrieve the velocity at each time:

```matlab
[Qt, Ut] = Exp_SE( U0, times );
```

`Log_SE` inverts the exponential — it recovers the algebra element from a group element:

```matlab
[dist, U] = Log_SE( Q );
% dist : geodesic distance from the identity
% U    : Lie-algebra element such that Exp_SE(U) ≈ Q
```

<Note>
`Log_SE` validates that `Q` is a valid SE(n) element (orthogonal rotation block, correct homogeneous structure). It will error if the input is malformed.
</Note>

### SIM(n) — similarity group

`Exp_SIM` handles the similarity group, whose Lie algebra elements combine a uniform scaling `sigma`, a skew-symmetric part `omega`, and a translation `tau`:

```matlab
% Build a SIM(2) algebra element: scale 0.1, rotation 0.3 rad, translation [1;2]
d = 2;
U0 = [ 0.1*eye(d) + [0, -0.3; 0.3, 0] , [1;2] ;
        zeros(1,d)                       ,  0   ];

Q1 = Exp_SIM( U0 );          % endpoint at t = 1
Qt = Exp_SIM( U0, 0:0.1:1 ); % path in 11 steps
```

Internally `Exp_SIM` decomposes the motion into an SO(n) part (via `Exp_SO`) and an ST(n) part (via `Exp_ST`), then composes them.

## Generic Exp — any matrix Lie group

`Exp` is the general-purpose Riemannian exponential. It integrates the geodesic equations numerically using a recursive Taylor expansion with adaptive step-size control.

### Signature

```matlab
Q1              = Exp( U0, B, MetricTensor )
[Q1, U1]        = Exp( U0, B, MetricTensor )
[Q1, U1, iters] = Exp( U0, B, MetricTensor )
Qt              = Exp( U0, B, MetricTensor, times )
Qt              = Exp( U0, B, MetricTensor, OPTIONS )
```

| Argument       | Description |
|----------------|-------------|
| `U0`           | Initial velocity — an `n×n` matrix in the Lie algebra, at the identity. |
| `B`            | Basis of the Lie algebra — an `(n²)×k` matrix (columns are vectorised basis elements). |
| `MetricTensor` | `k×k` symmetric positive-definite matrix defining the left-invariant metric on the algebra. |
| `times`        | Vector of non-negative, strictly increasing times at which to evaluate the geodesic (default `1`). |

### Starting from a non-identity base point

Because the metric is **left-invariant**, you translate the velocity to the identity, compute the exponential, and then left-multiply:

```matlab
% Geodesic starting at Q0 with velocity V0
Q1 = Q0 * Exp( Q0 \ V0, B, MetricTensor );
```

## Worked examples

The following examples are drawn directly from the `Exp.m` documentation comments.

### Example A — left-invariant geodesic on ST(1)

ST(1) is the group of 1D scalings and translations. Its Lie algebra has two basis elements.

```matlab
% Define the Lie algebra basis for ST(1)
B = [ 1 0 0 0 ; 0 0 1 0 ].';   % 4×2 matrix
MetricTensor = eye(2);

% Starting point and velocity
Q0 = [1 2; 0 1];
V0 = [-2 5; 0 0];

% Compute the geodesic endpoint
Q1 = Q0 * Exp( Q0 \ V0, B, MetricTensor );
```

### Example B — verifying left-invariance

The result should be zero (up to floating-point):

```matlab
Q0 * Exp( Q0 \ V0, B, MetricTensor ) - Q1
```

### Example C — tracing a geodesic path

```matlab
N = 1000;
Q = Q0;
V = V0 / N;
s = Q(1,1);
t = Q(1,2);

for j = 1:N
    [Q, V] = Exp( Q, V, B, MetricTensor );
    s(j+1) = Q(1,1);
    t(j+1) = Q(1,2);
end

figure;
plot( s, t );
axis equal;
```

### Example D — using CHECK\_Q

Supply a validation function that `Exp` calls after each integration step. If it returns `false`, the step size is reduced.

```matlab
check_ST1 = @(Q) isequal( [ abs(Q(1,1)), Q(1,2) ; 0, 1 ], Q );

Q1 = Exp( Q0 \ V0, B, MetricTensor, struct('CHECK_Q', check_ST1) );
```

### Example E — using PROJECT\_Q

Reprojection keeps `Q` on the group after each step:

```matlab
project_ST1 = @(Q) [ abs(Q(1,1)), Q(1,2) ; 0, 1 ];

Q1 = Exp( Q0 \ V0, B, MetricTensor, struct('PROJECT_Q', project_ST1) );
```

### Example F — metric scaling does not change the geodesic

Rescaling `MetricTensor` by any positive scalar leaves the geodesic path unchanged (only its parametrisation by arc-length changes):

```matlab
Exp( Q0 \ V0, B, MetricTensor ) - ...
Exp( Q0 \ V0, B, MetricTensor * 100 )
% Result is (approximately) zero
```

### Example G — metric conjugacy

If `C` is any invertible matrix, the geodesic transforms equivariantly under conjugation:

```matlab
C = [ 2 -3 ; 0 1 ];
MetricTensor_C = B.' * kron( inv(C), C.' ) * pinv(B).' * ...
                 MetricTensor                            * ...
                 pinv(B) * kron( inv(C).', C ) * B;

Exp( Q0 \ V0, B, MetricTensor_C ) - ...
inv(C) * Exp( C * (Q0 \ V0) * inv(C), B, MetricTensor ) * C
% Result is (approximately) zero
```

## Left-invariant vs right-invariant

`Exp` computes geodesics under a **left-invariant** metric. A right-invariant counterpart can be derived analytically.

<Tabs>
  <Tab title="Left-invariant (default)">
```matlab
% Exp is left-invariant by construction.
% Geodesic from Q0:
Q1 = Q0 * Exp( Q0 \ V0, B, MetricTensor );

% Check: left-multiplying by a constant H should give the same
% geodesic shape (translated by H)
H  = some_group_element;
Q1_shifted = H * Q0 * Exp( Q0 \ V0, B, MetricTensor );
% equals H * Q1
```
  </Tab>
  <Tab title="Right-invariant">
```matlab
% Wrap Exp to compute right-invariant geodesics:
Exp_right = @(Q, V, mB, MT, varargin) ...
    inv( Exp( inv(Q), -inv(Q)*V*inv(Q), mB, MT, varargin{:} ) );

Q1r = Exp_right( Q0, V0, B, MetricTensor );

% Verify right-invariance (result should be ~zero):
Exp_right( eye(2), V0 * inv(Q0), B, MetricTensor ) * Q0 - Q1r
```
  </Tab>
</Tabs>

## Integration options

Pass an `OPTIONS` struct as the fourth argument to `Exp` to control integration.

| Field | Default | Description |
|---|---|---|
| `T` | `[1]` | Times at which to evaluate the geodesic. |
| `ORDER` | auto | Taylor expansion order `R`. Auto-selected from `‖U0‖`. |
| `TOLERANCE` | `1e-10` | Maximum norm of the last Taylor term (used for step-size estimation). |
| `THRESHOLD_E` | `-1e-10` | Tolerance on the conserved energy `E`. Negative ⇒ relative. `0` ⇒ hard rescale. `Inf` ⇒ ignore. |
| `THRESHOLD_S` | `-1e-10` | Tolerance on the conserved momentum `S`. Negative ⇒ relative. `Inf` ⇒ ignore. |
| `ALPHA` | `0.75` | Step-size reduction factor. |
| `STEPSIZE` | `0` | Fixed step size (disables all adaptive controls). |
| `PROJECT_U` | `false` | Re-project the algebra element onto the algebra after each step. |
| `CHECK_Q` | `[]` | `@(Q)` function returning `true` if `Q` is on the group; triggers step reduction if `false`. |
| `PROJECT_Q` | `[]` | `@(Q)` function that re-projects `Q` onto the group after each step. |

```matlab
OPTS.T           = linspace(0, 1, 50);
OPTS.THRESHOLD_E = -1e-12;   % tight relative energy control
OPTS.ALPHA       = 0.8;

Qt = Exp( U0, B, MetricTensor, OPTS );
```

<Warning>
All `OPTIONS` field names must be **upper-case**. Unknown fields trigger a warning but are otherwise ignored.
</Warning>

## Pre-built algebra bases

Common bases are listed in the comments of `Exp.m` and can also be obtained programmatically via `LIEbasis`.

<CodeGroup>

```matlab 2D groups
B_SO2  = LIEbasis('so(2)');
B_SE2  = LIEbasis('se(2)');
B_SIM2 = LIEbasis('sim(2)');
B_ST1  = LIEbasis('st(1)');
B_ST2  = LIEbasis('st(2)');
```

```matlab 3D groups
B_SO3  = LIEbasis('so(3)');
```

```matlab General linear groups
B_GL2 = LIEbasis('gl(2)');
```
